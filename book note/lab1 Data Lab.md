*********************
2. 用btest测试
*********************

这个目录中的Makefile编译了你的bits.c版本，并加上了
额外的代码来创建一个名为btest的程序（或测试线束）。

要编译和运行btest程序，请键入。

    unix> make btest
    unix> ./btest [可选的cmd行参数] 。

你需要在每次改变 bits.c
程序时都需要重新编译btest。当从一个平台转移到另一个平台时，你会想要
摆脱旧版本的 btest 并生成一个新版本。 使用
命令。

    unix> make clean
    unix> make btest

Btest通过对每个函数运行数以百万计的测试案例来测试你的代码是否正确。
来测试你的代码的正确性，在每个函数上运行数百万个测试案例。 它测试的范围很广，围绕着众所周知的角
的角落，如整数谜题的Tmin和0，以及0、inf和
浮点数谜题中的Tmin和零，以及零、inf和去正态化和正态化数字之间的边界。
点谜题。当btest在你的某个函数中检测到一个错误时。
它将打印出失败的测试，错误的结果，以及预期的结果。
然后终止对该函数的测试。

下面是btest的命令行选项。

  unix> ./btest -h
  使用方法。./btest [-hg] [-r <n>] [-f <name> [-1|-2|-3 <val>]*] [-T <time limit>] [-t [-T <时间限制>]。
    -1 <val> 指定第一个函数参数
    -2 <val> 指定第二个函数参数
    -3 <值> 指定第三个函数参数
    -f <名称> 只测试指定的函数
    -g 自动交易的输出格式，没有错误信息
    -h 打印此信息
    -r <n> 对所有问题给出统一的权重n
    -T <lim> 设置超时限制为lim

例子。

  测试所有函数的正确性并打印出错误信息。
  unix> ./btest

  以紧凑的形式测试所有函数，没有错误信息。
  unix> ./btest -g

  测试函数 foo 的正确性。
  unix> ./btest -f foo

  用特定的参数测试函数foo的正确性。
  unix> ./btest -f foo -1 27 -2 0xf

Btest不会检查你的代码是否符合编码准则。
